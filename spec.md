# 功能規格書（Specification）

> 本文件描述 Meeting Minutes AI 的核心功能需求與使用者故事。
> 專注於「做什麼」與「為何這樣做」，不涉及實作細節。

---

## 目標使用者

需要快速將會議錄音轉化為結構化紀錄的團隊成員、專案經理、或個人工作者。

---

## 核心 AI 引擎

- **架構：** 策略模式（Strategy Pattern），透過抽象 `AudioProcessor` 介面實現模型抽換
- **雙軌 Provider：**
  - **GeminiProvider（遠端）：** 利用 Gemini 原生多模態能力，單一 Prompt 同時產出逐字稿、摘要與 Action Items
  - **LocalWhisperProvider（本地端）：** 使用本地 Whisper 模型轉出逐字稿，可選配 Ollama 提供本地摘要
- **切換機制：** 透過環境變數 `MODEL_MODE=remote|local` 或前端介面動態選擇 Provider
- **輸出語言：** 遠端模式統一輸出**繁體中文**；本地端模式逐字稿保留原始語言
- **健康檢查：** 啟動時偵測 Provider 可用性，執行時再次確認

## 功能一：檔案上傳

### 使用者故事

> 身為一位**會議參與者**，我希望能**上傳會議錄音與相關標註檔案**，以便後續進行語音辨識、角色辨識與摘要。

### 需求描述

| 項目 | 說明 |
|------|------|
| **音檔格式** | `.mp3`、`.wav`、`.flac` |
| **標註檔格式** | `.TextGrid`（語音標註/逐字稿/音韻分析）、`.rttm`（角色辨識） |
| 檔案大小限制 | 單檔上限 300MB（可配置） |
| 上傳方式 | 透過 Web 介面選擇檔案上傳，音檔為必要，標註檔為選填 |
| 上傳回饋 | 顯示上傳進度條，上傳完成後顯示檔案資訊（名稱、大小、時長） |
| 格式驗證 | 上傳時檢查檔案格式，不支援的格式需顯示明確錯誤訊息 |
| 會議標題 | **選填**，使用者未填寫時由 AI 根據摘要自動命名 |

### 支援檔案類型說明

| 檔案類型 | 副檔名 | 用途 | 必要性 |
|----------|--------|------|--------|
| 音檔 | `.mp3` `.wav` `.flac` | 會議錄音，作為 AI 處理的主要輸入 | **必要**（擇一） |
| TextGrid | `.TextGrid` | Praat 語音標註檔，可匯入語音標註、逐字稿、音韻分析結果 | 選填 |
| RTTM | `.rttm` | Rich Transcription Time Marked 檔案，匯入說話者角色辨識（Speaker Diarization）結果 | 選填 |

### 標註檔匯入行為

- **TextGrid 匯入：** 解析 TextGrid 中的 interval tiers，將標註內容合併至逐字稿，保留時間戳記與標籤資訊
- **RTTM 匯入：** 解析 RTTM 中的說話者片段，為逐字稿標示「誰在什麼時間說了什麼」
- **同時匯入：** 若同時上傳 TextGrid + RTTM，系統將交叉比對時間軸，產出帶有角色標籤的逐字稿
- **僅上傳標註檔（無音檔）：** 不允許，音檔為必要輸入
- **標註檔與 AI 處理的關係：** 若已匯入 TextGrid 逐字稿，AI 處理時可跳過轉錄步驟，直接使用匯入的逐字稿進行摘要

### 驗收條件（Acceptance Criteria）

- [ ] 使用者可成功上傳 `.mp3`、`.wav`、`.flac` 格式的音檔
- [ ] 使用者可選擇性附加上傳 `.TextGrid` 檔案
- [ ] 使用者可選擇性附加上傳 `.rttm` 檔案
- [ ] 上傳超過大小限制的檔案時，顯示友善錯誤提示
- [ ] 上傳非支援格式的檔案時，顯示友善錯誤提示
- [ ] 上傳完成後顯示檔案基本資訊
- [ ] 會議標題為選填，未填時 AI 自動生成
- [ ] TextGrid 匯入後，逐字稿區域顯示解析後的標註內容
- [ ] RTTM 匯入後，逐字稿中標示說話者角色
- [ ] 僅上傳標註檔（無音檔）時，顯示錯誤提示

## 功能二：一鍵 AI 處理（轉錄 + 摘要 + Action Items）

### 使用者故事

> 身為一位**會議參與者**，我希望**一鍵就能完成所有處理**，不需要分步操作，系統自動產出逐字稿、摘要與待辦事項。

### 需求描述

| 項目 | 說明 |
|------|------|
| 觸發方式 | 使用者選擇音檔後按下「開始」，系統自動完成上傳與處理（一步驟） |
| 處理方式 | 依選擇的 Provider 處理：遠端（Gemini 一次產出全部）或本地端（Whisper 逐字稿）。若已匯入 TextGrid 逐字稿，可跳過轉錄直接摘要 |
| Provider 選擇 | 透過環境變數預設，前端可覆寫選擇 |
| 語言支援 | 可辨識任何語言的語音；遠端模式輸出繁體中文，本地模式保留原始語言 |
| 並行限制 | MVP 階段**不允許**同時處理多個音檔，採排隊機制 |
| 處理狀態 | 提供即時狀態顯示（處理中 / 完成 / 失敗） |

### 輸出內容

#### 逐字稿
- 段落式時間戳記格式，以發言段落為單位標示時間
- 若有 RTTM 角色辨識資料，在時間戳記後加上說話者標籤
- 範例（無角色辨識）：
  ```
  [00:00 - 00:32] 大家好，今天的會議主題是專案進度追蹤，我們先來看上週的進度報告...
  [00:32 - 01:15] 後端的部分，API 已經完成了 80%，目前還在處理錯誤處理的部分...
  ```
- 範例（有 RTTM 角色辨識）：
  ```
  [00:00 - 00:32] [Speaker_1] 大家好，今天的會議主題是專案進度追蹤...
  [00:32 - 01:15] [Speaker_2] 後端的部分，API 已經完成了 80%...
  ```

#### 會議摘要
- 結構化摘要，包含：會議主題、重點討論事項、決議事項
- 統一輸出繁體中文

#### Action Items（待辦事項）
- 每項包含：任務描述、負責人（若可辨識）、截止日期（若有提及）

### 驗收條件（Acceptance Criteria）

- [ ] 一鍵觸發後自動完成轉錄、摘要、Action Items 提取
- [ ] 逐字稿以段落式時間戳記呈現
- [ ] 摘要包含會議主題、重點討論、決議事項
- [ ] Action Items 包含任務描述，負責人與截止日期為選填
- [ ] 處理過程中顯示狀態與使用的 Provider 名稱
- [ ] 處理失敗時依錯誤類型顯示對應提示（429 提示切換本地模式、401 提示檢查 API Key）
- [ ] 使用者可**手動點擊重試**，重試時允許切換 Provider
- [ ] 同一時間僅處理一個音檔
- [ ] 使用者可在前端選擇處理模式（遠端 / 本地端）
- [ ] 本地端模式未配置 Ollama 時，隱藏摘要區塊並顯示「配置 Ollama 以啟用本地摘要」

## 功能三：結果查看與編輯

### 使用者故事

> 身為一位**專案經理**，我希望能在網頁上**查看並修改** AI 產出的逐字稿、摘要與待辦事項，確保內容正確後作為正式紀錄。

### 需求描述

| 項目 | 說明 |
|------|------|
| 顯示內容 | 逐字稿、會議摘要、Action Items 分區顯示 |
| 編輯方式 | 支援 **Markdown 純文字編輯**（逐字稿與摘要） |
| Action Items 操作 | 可修改描述/負責人/截止日期、可新增/刪除項目、可標記完成 |
| 儲存方式 | 手動點擊「儲存」按鈕儲存修改 |

### 驗收條件（Acceptance Criteria）

- [ ] 結果頁分區顯示逐字稿、摘要、Action Items
- [ ] 使用者可編輯逐字稿與摘要（Markdown 純文字）
- [ ] 使用者可新增、修改、刪除 Action Items
- [ ] 使用者可標記 Action Items 完成狀態
- [ ] 修改後點擊儲存按鈕生效

## 功能四：Web 介面與歷史紀錄

### 使用者故事

> 身為一位**使用者**，我希望有一個**簡單的網頁介面**來上傳錄音、查看結果，並能瀏覽過去的會議紀錄。

### 需求描述

| 項目 | 說明 |
|------|------|
| 介面風格 | 簡潔直覺，操作流程清晰 |
| 主要頁面 | **首頁（上傳頁）**：上傳音檔的入口 |
|  | **結果頁**：顯示逐字稿、會議摘要、Action Items |
|  | **歷史紀錄頁**：列出過去的會議紀錄 |
| 歷史紀錄 | 依建立時間**倒序**排列，MVP 不需搜尋與分頁 |
| 刪除功能 | 可刪除會議紀錄，使用瀏覽器 `confirm` 對話框確認 |
| 音檔管理 | 處理完成後，使用者可**手動刪除音檔**以節省空間，文字紀錄保留 |
| 響應式設計 | 支援桌面與行動裝置瀏覽 |
| 無需登入（MVP） | 初版不需使用者認證 |

### 驗收條件（Acceptance Criteria）

- [ ] 使用者可透過網頁上傳音檔
- [ ] 上傳後可在結果頁查看逐字稿、摘要與 Action Items
- [ ] 歷史紀錄頁依時間倒序列出過去的會議紀錄
- [ ] 使用者可刪除會議紀錄
- [ ] 使用者可單獨刪除已處理完的音檔
- [ ] 介面在桌面與手機瀏覽器上均可正常顯示

```

│  + 選擇 Provider → 按下「開始」               │     │             │

│                                              │     │             │
│  ┌─ 有 TextGrid ──────────────────────────┐  │     │             │
│  │ 使用者選擇：                            │  │     │             │
│  │  ○ 使用 TextGrid 逐字稿 → 跳過轉錄     │  │     │             │
│  │  ○ 重新 AI 轉錄 → 忽略 TextGrid        │  │     │             │
│  └────────────────────────────────────────┘  │     │             │
│  ┌─ 無 TextGrid → remote ────────────────┐  │     │             │
│  │ Gemini（逐字稿 + 摘要 + Actions）       │  │     │             │
│  └────────────────────────────────────────┘  │     │             │
│  ┌─ 無 TextGrid → local ─────────────────┐  │     │             │
│  │ Whisper 逐字稿 + Ollama 摘要（若配置）   │  │     │             │
│  └────────────────────────────────────────┘  │     │             │
│  ┌─ 有 RTTM ─────────────────────────────┐  │     │             │
│  │ 逐字稿加上說話者角色標籤                 │  │     │             │
│  └────────────────────────────────────────┘  │     │             │
└─────────────────────────────────────────────┘     └─────────────┘
```

## 非功能需求

| 項目 | 說明 |
|------|------|
| **效能** | 10 分鐘音檔應在合理時間內完成處理 |
| **可靠性** | 處理失敗時不影響已儲存的資料，支援手動重試 |
| **可擴充性** | 策略模式架構，允許新增 Provider（如 Azure Speech）且不影響現有程式碼 |
| **資料保留** | 文字紀錄持久化儲存；音檔可由使用者手動刪除 |

## 未來擴充方向（不在 MVP 範圍）

- LocalWhisperProvider 串接 Ollama 提供本地摘要（架構已預留，可選配）
- 新增更多 Provider（Azure Speech、AWS Transcribe 等）
- 使用者認證與權限管理
- 自動說話者辨識（Speaker Diarization，目前僅支援匯入 RTTM）
- 多語言輸出選擇
- 會議紀錄匯出（PDF / Markdown）
- 即時錄音（透過麥克風直接錄製）
- 與行事曆或專案管理工具整合
- 搜尋與分頁功能
- 並行處理多個音檔
