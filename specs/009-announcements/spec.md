# 功能規格：最新消息 / 公告訊息

## 概述

提供系統內建的公告訊息功能，讓管理員能發布重要消息給使用者，使用者登入後能即時看到最新公告。用於傳達系統更新、維護通知、政策變更等資訊。

## 使用者角色

| 角色 | 能做的事 |
|------|---------|
| 超級管理員（Role 1） | 建立、編輯、刪除所有公告；置頂/取消置頂 |
| 管理員（Role 2） | 建立、編輯、刪除自己的公告 |
| 一般使用者（Role 3） | 查看公告列表、查看公告詳情 |

## 使用者情境與驗收場景

### US-01：管理員發布公告（P1）

**身為**管理員，**我想要**發布公告訊息，**以便**通知使用者重要資訊。

**驗收場景：**

- **Given** 管理員已登入且角色為 Role 1 或 Role 2
- **When** 管理員填寫公告標題與內容並送出
- **Then** 公告成功建立，所有使用者可在公告列表中看到

### US-02：使用者查看公告列表（P1）

**身為**使用者，**我想要**看到所有公告的列表，**以便**瀏覽最新消息。

**驗收場景：**

- **Given** 使用者已登入
- **When** 使用者點擊導覽列的「最新消息」
- **Then** 顯示公告列表，依建立時間倒序排列，置頂公告排最前面

### US-03：使用者查看公告詳情（P1）

**身為**使用者，**我想要**點擊公告查看完整內容，**以便**了解公告細節。

**驗收場景：**

- **Given** 使用者已登入且在公告列表頁
- **When** 使用者點擊某一則公告
- **Then** 顯示公告完整標題、內容、發布者名稱、發布時間

### US-04：管理員編輯公告（P2）

**身為**管理員，**我想要**編輯已發布的公告，**以便**修正內容或更新資訊。

**驗收場景：**

- **Given** 管理員已登入且為該公告的發布者（或超級管理員）
- **When** 管理員修改公告內容並儲存
- **Then** 公告內容更新，顯示最後更新時間

### US-05：管理員刪除公告（P2）

**身為**管理員，**我想要**刪除不再需要的公告，**以便**保持公告列表整潔。

**驗收場景：**

- **Given** 管理員已登入且為該公告的發布者（或超級管理員）
- **When** 管理員確認刪除
- **Then** 公告從列表中移除

### US-06：超級管理員置頂公告（P2）

**身為**超級管理員，**我想要**將重要公告置頂，**以便**確保所有人優先看到。

**驗收場景：**

- **Given** 超級管理員已登入
- **When** 超級管理員將某公告設為置頂
- **Then** 該公告在列表中永遠排在最前面（多則置頂依時間倒序）

## Clarifications

### Session 2026-03-31

- Q: 公告標題與內容的長度限制？ → A: 標題 255 字元、內容 5,000 字元（標準型）
- Q: 公告列表為空時的顯示方式？ → A: 顯示「目前沒有公告」提示文字，管理員額外顯示「發布第一則公告」按鈕
- Q: 搜尋範圍？ → A: 僅搜尋標題（前端 JS 篩選）

## 功能需求

### FR-01：公告列表頁

- 導覽列新增「最新消息」連結，所有登入使用者可見
- 列表顯示：標題、發布者名稱、發布時間、是否置頂
- 排序規則：置頂公告排最前 → 其餘依建立時間倒序
- 支援搜尋（關鍵字搜尋標題）
- 空狀態：顯示「目前沒有公告」提示文字；管理員（Role 1、2）額外顯示「發布第一則公告」按鈕

### FR-02：公告詳情頁

- 顯示完整標題、內容、發布者名稱、發布時間
- 若公告已被編輯過，顯示「最後更新時間」
- 管理員可見「編輯」和「刪除」按鈕（依權限）

### FR-03：建立公告

- 表單欄位：標題（必填，最長 255 字元）、內容（必填，最長 5,000 字元）
- 僅 Role 1、Role 2 可使用
- 建立後跳轉到公告列表

### FR-04：編輯公告

- 發布者本人或超級管理員可編輯
- 可修改標題與內容
- 儲存後更新 `updated_at` 時間

### FR-05：刪除公告

- 發布者本人或超級管理員可刪除
- 刪除前需確認對話框
- 刪除後跳轉到公告列表

### FR-06：置頂功能

- 僅超級管理員可操作
- 可設定/取消置頂
- 置頂公告在列表中排最前面

## 關鍵實體

### Announcement（公告）

| 欄位 | 說明 |
|------|------|
| id | 唯一識別碼（UUID） |
| title | 標題（最長 255 字元） |
| content | 內容（最長 5,000 字元） |
| is_pinned | 是否置頂 |
| created_by | 發布者 ID（關聯 User） |
| created_at | 建立時間 |
| updated_at | 最後更新時間 |

## 成功標準

- 使用者登入後 2 次點擊內可看到公告列表
- 管理員 1 分鐘內可完成一則公告的發布
- 公告發布後所有使用者立即可見
- 置頂公告永遠排在列表最前面

## 範圍外（不做）

- 公告的附件上傳功能
- 公告的已讀/未讀追蹤
- 公告的推播通知
- 公告的排程發布（定時上線）
- 公告的分類/標籤系統

## 假設

- 公告數量不會太大（數百筆等級），前端篩選即可，不需後端分頁
- 公告內容為純文字，不需富文字編輯器
- 所有登入使用者都能看到所有公告（公告沒有可見性分級）

## 依賴

- 現有的三級角色權限系統（Role 1/2/3）
- 現有的認證機制（Session + Cookie）
- 現有的 User 資料模型
