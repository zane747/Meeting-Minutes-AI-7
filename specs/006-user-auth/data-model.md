# Data Model: 使用者登入登出系統

> 本文件定義此功能涉及的資料實體、欄位、關聯與驗證規則。
> 【新手導讀】資料模型就像「表格的設計圖」——定義資料庫裡要存什麼、怎麼存。

---

## 實體關係概覽

> **術語解釋**：「實體」就是資料庫裡的一張表（table）。
> 一個實體包含多個「欄位」（fields），每個欄位存一種資料。

```
┌─────────────────────┐
│       User          │
│─────────────────────│
│ id          (PK)    │  ← PK = Primary Key（主鍵），每筆資料的唯一身份證
│ username    (UQ)    │  ← UQ = Unique（唯一），不能重複
│ password_hash       │  ← 密碼的雜湊值（不存原始密碼！）
│ is_active           │  ← 帳號是否啟用
│ created_at          │
│ updated_at          │
└─────────────────────┘
         │
         │ 1（一個使用者）
         │
         ▼
         ╳ 目前不建立與 Meeting 的關聯
           （v1 所有使用者可見所有會議）
```

## 實體定義

### User（使用者）

> 這張表存放所有註冊使用者的資訊。
> 注意：密碼**絕對不會**以原始文字存入，只存雜湊後的結果。

| 欄位 | 型別 | 限制 | 說明 |
|------|------|------|------|
| `id` | String(36) | PK, 自動生成 UUID | 每個使用者的唯一識別碼 |
| `username` | String(30) | NOT NULL, UNIQUE, 3~30 字元 | 帳號名稱（英數字與底線），同時作為顯示名稱 |
| `password_hash` | String(255) | NOT NULL | bcrypt 雜湊後的密碼 |
| `is_active` | Boolean | NOT NULL, 預設 True | 帳號是否啟用（為未來停權功能預留） |
| `created_at` | DateTime | NOT NULL, 自動填入 | 帳號建立時間 |
| `updated_at` | DateTime | NOT NULL, 自動更新 | 最後更新時間 |

### 欄位驗證規則

> **術語解釋**：驗證（Validation）就是「檢查使用者輸入的資料是否合格」。
> 例如：帳號不能太短、密碼不能太簡單。這些檢查在資料存入資料庫之前就會做。

| 規則 | 描述 | 對應需求 |
|------|------|---------|
| 帳號格式 | 正則表達式 `^[a-zA-Z0-9_]{3,30}$`（只允許英文字母、數字、底線，3~30 字） | FR-010 |
| 帳號唯一 | 資料庫層級 UNIQUE 約束 + 應用層提前檢查 | FR-002 |
| 密碼長度 | 最少 8 字元（在應用層驗證，雜湊前檢查） | FR-011 |
| 密碼儲存 | 必須經過 bcrypt 雜湊，永不明文存入 | FR-003 |

### 索引（Index）

> **術語解釋**：索引就像書的「目錄」。
> 沒有索引時，資料庫要從頭到尾找（像翻整本書找某個詞）。
> 有了索引，資料庫可以直接跳到正確的位置（像用目錄找到頁碼）。

| 索引名稱 | 欄位 | 類型 | 用途 |
|----------|------|------|------|
| `ix_users_username` | `username` | UNIQUE | 加速登入時的帳號查詢 |

## Session 管理（不建表）

> **為什麼 Session 不需要建資料庫表？**
> 因為我們使用 `starlette-session`，它會把 session 資料存在伺服器記憶體中，
> 並透過 Cookie 裡的 session ID 來對應。
>
> 這種做法在單機部署（我們的情況）下最簡單有效。
> 缺點是：如果伺服器重新啟動，所有人都要重新登入。
> 但對內部工具來說，這完全可以接受。

**Session 資料結構（存在記憶體中）**:

| 鍵（Key） | 值（Value） | 說明 |
|-----------|-------------|------|
| `user_id` | String | 當前登入使用者的 ID |
| `username` | String | 當前登入使用者的帳號（方便模板直接使用） |

**Cookie 設定**:

| 設定 | 值 | 說明 |
|------|-----|------|
| Cookie 名稱 | `session` | 瀏覽器中的 Cookie 名稱 |
| `httponly` | True | JavaScript 無法讀取此 Cookie（防 XSS 攻擊） |
| `max_age` | 86400（24小時） | Cookie 過期時間，對應 session 有效期 |
| `samesite` | `lax` | 防止 CSRF 攻擊的基本保護 |
