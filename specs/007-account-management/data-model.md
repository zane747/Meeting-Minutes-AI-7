# Data Model: 帳號管理

> 此功能不新增資料表，沿用 006-user-auth 的 User 表。

---

## 沿用的實體

### User（使用者）— 已存在

| 欄位 | 型別 | 此功能如何使用 |
|------|------|---------------|
| `id` | String(36) | 識別要操作的帳號 |
| `username` | String(30) | 列表顯示、識別帳號 |
| `password_hash` | String(255) | 修改密碼時更新此欄位 |
| `is_active` | Boolean | **核心欄位**：停用/啟用操作直接改這個值 |
| `created_at` | DateTime | 列表顯示建立時間 |
| `updated_at` | DateTime | 自動更新 |

### 狀態轉換圖

```
         建立帳號
            │
            ▼
    ┌──────────────┐
    │  啟用 (True)  │ ←──── 啟用操作
    └──────┬───────┘
           │
       停用操作
           │
           ▼
    ┌──────────────┐
    │  停用 (False) │
    └──────┬───────┘
           │
       刪除操作
           │
           ▼
       從資料庫移除
```

### 保護規則

- 不能停用/刪除自己（比對 session 中的 user_id 和操作目標的 id）
- 至少保留一個 is_active=True 的帳號（操作前 COUNT 查詢）
